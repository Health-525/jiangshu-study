# 程序员最常见的 30 个动词：读懂报错/工单的关键

很多同学学英语只背名词（server、database、cache），但真正决定你能不能读懂日志、PR、工单的，是**动词**。

下面这 30 个动词，基本覆盖了「开发—上线—故障—修复」的日常沟通。

---

## 1）排查 & 解释问题（Troubleshooting）
- **debug**：调试
- **investigate**：调查（更正式）
- **trace**：追踪（调用链/链路）
- **reproduce**：复现
- **isolate**：隔离（缩小范围）
- **identify**：定位/识别
- **diagnose**：诊断（偏“判断原因”）
- **explain**：解释

**例句**
- We **reproduced** the issue on staging.
- Let’s **isolate** the failing component first.

## 2）修复 & 改动（Fix & Change）
- **fix**：修复
- **patch**：打补丁（快速修）
- **refactor**：重构（不改外部行为）
- **optimize**：优化
- **rewrite**：重写
- **clean up**：清理（代码/数据）
- **remove**：移除
- **rename**：重命名

**例句**
- I’ll **refactor** this module to improve readability.
- We should **remove** dead code.

## 3）发布 & 运维（Release & Ops）
- **build**：构建
- **deploy**：部署
- **release**：发布
- **rollback**：回滚
- **restart**：重启
- **scale**：扩缩容
- **migrate**：迁移（DB/服务/版本）
- **monitor**：监控

**例句**
- We **deployed** a hotfix and **monitored** the metrics.
- If errors spike, we’ll **rollback**.

## 4）可靠性/兼容性（Reliability）
- **retry**：重试
- **throttle**：限流
- **degrade**：降级
- **recover**：恢复
- **fail**：失败
- **break**：破坏/导致不可用（也指 breaking change）

**例句**
- The request **timed out**, so the client will **retry**.
- This change may **break** backward compatibility.

## 5）版本 & 规范（Versioning）
- **deprecate**：弃用（还能用，但不建议）
- **support**：支持（版本/平台）
- **enforce**：强制执行（规则/校验）

**例句**
- This API is **deprecated**; please **migrate** to v2.
