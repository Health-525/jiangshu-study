# PR / 代码评审常用英语（写得像个老外同事）

代码评审的核心不是“英语多高级”，而是：**礼貌、明确、可执行**。

这篇给一套能直接复制粘贴的句型库。

---

## 1）正向反馈（先肯定，再建议）
- **LGTM**. Nice improvement.
- Looks good overall.
- Great catch / Good point.
- Thanks for addressing the previous comments.

## 2）请求补充（不强硬，但清晰）
- Could we add a test for this case?
- Can you clarify the **rationale** behind this change?
- Do we have metrics showing this improves performance?
- Could you add a short comment explaining why?

## 3）小建议（Nit）
- **Nit**: rename this variable for clarity.
- **Nit**: minor formatting / typo.
- This could be simplified a bit.

> “Nit”=小问题/非阻塞建议，语气比“这是错的”柔和很多。

## 4）担忧 & 风险提示（非常实用）
- This might introduce a **breaking change**.
- I’m concerned about edge cases when …
- What happens if the upstream service is slow/unavailable?
- Could this cause data loss?

## 5）必须修改（Blocking / Required）
- This is **blocking** for me.
- We should fix this before merging.
- Please handle the null/empty case.
- This needs to be addressed to avoid a security issue.

## 6）收尾动作
- Once these are addressed, I’m happy to approve.
- After the update, please re-request review.
- Let’s merge after CI is green.

---

## 7）一句话总结：写评审的“黄金结构”
1) 先肯定（1 句）
2) 给建议（可执行）
3) 标明是否阻塞（nit / optional / required）
