# 系统设计高频词：吞吐、延迟、可用性到底怎么说

做系统设计/性能优化时，中文里我们习惯说“吞吐、延迟、可用性”。

英文里对应的不只是翻译，更有一套常见搭配（collocations）。背对了，你的表达就会像机器翻译。

---

## 1）性能三件套（Performance Trio）

### **throughput**（吞吐量）
- 常见搭配：increase throughput / throughput bottleneck
- 例句：We need to increase **throughput** during peak hours.

### **latency**（延迟）
- 常见搭配：reduce latency / network latency / database latency
- 例句：This change reduces **latency** by 20%.

### **tail latency**（尾延迟）
- 指 p95/p99 这种“最慢的一批请求”。
- 例句：We care more about **p99 latency** than average latency.

---

## 2）可靠性相关（Reliability）

### **availability**（可用性）
- 例句：The goal is 99.9% **availability**.

### **durability**（持久性/数据不丢）
- 常见语境：存储、数据库
- 例句：The system guarantees data **durability**.

### **fault tolerance**（容错）
- 例句：We improved **fault tolerance** by adding redundancy.

---

## 3）一致性/分布式常见词
- **consistency**：一致性
- **partition**：网络分区
- **replication**：复制
- **trade-off**：权衡

**例句**
- This improves **availability** but weakens **consistency**.
- We need to discuss the **trade-offs**.

---

## 4）一句话模板（面试/汇报很有用）
- The bottleneck is ____ , so we propose ____ to improve ____.
- The main trade-off is ____ vs ____.
- We optimized for ____ under the constraint of ____.
